
# API Authentication Guide for SolFoundry

## Overview
This guide provides developers with information on how to securely generate and use API keys or JWT (JSON Web Tokens) for authenticating with the SolFoundry backend.

---

## API Key Authentication

### Generating API Keys
1. **Access the Admin Panel**: Log in to the SolFoundry admin interface.
2. **Navigate to API Keys Section**: Go to the "API Keys" tab in the admin dashboard.
3. **Generate a New Key**:
   - Click on "Generate New Key"
   - Provide a descriptive name for the key (e.g., "Production API Key")
   - Set an expiration date if needed
   - Click "Generate"

### Using API Keys
API keys are typically used in the `Authorization` header of HTTP requests:

```http
Authorization: Bearer YOUR_API_KEY_HERE
```

### Best Practices for API Keys
- **Store securely**: Never hardcode API keys in client-side code. Use environment variables or secure configuration management.
- **Limit permissions**: Create keys with the least privileges required for the application.
- **Rotate keys**: Regularly rotate API keys, especially if they are compromised or no longer needed.
- **Monitor usage**: Track API key usage to detect unauthorized access.

## JWT Authentication

### Generating JWT Tokens
JWT tokens are typically generated by the authentication endpoint of the SolFoundry backend.

#### Example Request
```bash
curl -X POST \
  http://your-solfoundry-instance.com/auth/login \
  -H 'Content-Type: application/json' \
  -d '{
    "username": "your_username",
    "password": "your_password"
  }'
```

#### Response
A successful response will include a JWT token:

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c"
}
```

### Using JWT Tokens
Include the JWT token in the `Authorization` header of subsequent requests:

```http
Authorization: Bearer YOUR_JWT_TOKEN_HERE
```

### Best Practices for JWT Tokens
- **Short-lived tokens**: Use short-lived access tokens and refresh tokens when needed.
- **Secure storage**: Store tokens securely in HTTP-only cookies or secure storage mechanisms.
- **HTTPS**: Always use HTTPS to prevent token interception.
- **Token revocation**: Implement a mechanism to revoke tokens if compromised.

## API Key vs JWT Comparison

| Feature          | API Key                          | JWT Token                        |
|------------------|----------------------------------|----------------------------------|
| **Use Case**     | Simple, long-lived authentication | Short-lived, role-based access   |
| **Security**     | Less secure if exposed          | More secure with proper setup    |
| **Flexibility**  | Limited to key-based auth        | Supports claims and scopes       |
| **Rotation**     | Manual key rotation             | Automatic token expiration       |

## Troubleshooting

### Common Issues
- **Invalid Key/Token**: Ensure the key or token is correctly generated and not expired.
- **Permissions Denied**: Verify that the key or token has the necessary permissions.
- **Network Issues**: Check if the request is reaching the server and if there are any network restrictions.

### Debugging Tips
- Use tools like `curl` or Postman to test API requests.
- Check server logs for detailed error messages.
- Ensure the server is running and accessible.
