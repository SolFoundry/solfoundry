# Good First Issue Guide

## Introduction

Welcome to SolFoundry! This guide is designed to help new developers get started with our project by contributing to issues labeled as "good first issue."

---

## Finding Good First Issues

1. **Visit the Issues Page**:
   - Go to the [SolFoundry GitHub Issues](https://github.com/SolFoundry/solfoundry/issues).

2. **Filter by Label**:
   - Use the label filter to find issues labeled "good first issue".
   - Click on the "Labels" filter on the left sidebar and select "good first issue".

3. **Review the Issues**:
   - Look through the list of issues to find something that interests you.
   - These issues are designed to be beginner-friendly and help you get familiar with the codebase.

---

## Forking the Repository

1. **Fork the Repository**:
   - Click on the "Fork" button at the top right corner of the [SolFoundry GitHub Repository](https://github.com/SolFoundry/solfoundry).

2. **Clone Your Fork**:
   - Copy the URL of your forked repository.
   - Open your terminal and clone the repository using:
     ```bash
     git clone https://github.com/your-username/solfoundry.git
     ```
   - Navigate into the cloned repository:
     ```bash
     cd solfoundry
     ```

---

## Setting Up Your Development Environment

1. **Install Dependencies**:
   - Ensure you have Python and pip installed.
   - Install the project dependencies:
     ```bash
     pip install -r requirements.txt
     ```

2. **Create a New Branch**:
   - Create a new branch for your changes:
     ```bash
     git checkout -b your-branch-name
     ```

---

## Making Your Contribution

1. **Select an Issue**:
   - Pick an issue from the "good first issue" list.
   - Read the issue description thoroughly.

2. **Implement Changes**:
   - Make the necessary changes to implement the issue.
   - Follow the project's coding standards and style.

3. **Test Your Changes**:
   - Run the tests to ensure your changes do not break existing functionality:
     ```bash
     pytest
     ```

---

## Submitting a Pull Request

1. **Commit Your Changes**:
   - Commit your changes with a descriptive message:
     ```bash
     git add .
     git commit -m "Describe your changes here"
     ```

2. **Push Your Changes**:
   - Push your changes to your forked repository:
     ```bash
     git push origin your-branch-name
     ```

3. **Create a Pull Request**:
   - Go to your forked repository on GitHub.
   - Click on "Compare & pull request" to create a new pull request.
   - Fill in the PR description with details about the changes you made.
   - Submit the pull request.

---
