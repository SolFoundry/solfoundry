# SolFoundry Brand Guide

> The definitive guide to SolFoundry's visual identity, tone, and design principles.

---

## 1. Brand Overview

**SolFoundry** is a bounty platform where code meets reward. The brand evokes a digital forge — industrial precision, craft, and value creation through open-source contribution.

### Brand Personality
- **Precise** — We value craft and quality
- **Open** — Transparent, community-driven, meritocratic
- **Bold** — Confident in rewarding builders
- **Technical** — Built by developers, for developers

### Tagline
*"Forge code. Earn rewards."*

## 3. Typography

### Font Families

| Font | Family | Usage |
|------|--------|-------|
| **Orbitron** | Display | Headlines, hero text, brand name |
| **Inter** | Sans-serif | Body text, UI elements, buttons |
| **JetBrains Mono** | Monospace | Code blocks, terminal output, reward amounts |

### Type Scale

| Element | Font | Size | Weight | Line Height |
|---------|------|------|--------|-------------|
| Hero Headline | Orbitron | 48px / `text-5xl` | 700 (Bold) | 1.1 |
| Page Title | Orbitron | 36px / `text-4xl` | 700 | 1.2 |
| Section Heading | Inter | 24px / `text-2xl` | 600 (Semibold) | 1.3 |
| Card Title | Inter | 18px / `text-lg` | 600 | 1.4 |
| Body | Inter | 14px / `text-sm` | 400 (Regular) | 1.6 |
| Small / Labels | Inter | 12px / `text-xs` | 500 (Medium) | 1.4 |
| Code | JetBrains Mono | 13px | 400 | 1.5 |
| Reward Amount | JetBrains Mono | 16px | 700 | 1.2 |

### Typography Rules
- ✅ **Do**: Use Orbitron sparingly — only for hero text and the brand name
- ✅ **Do**: Use Inter for all UI text and body copy
- ✅ **Do**: Use JetBrains Mono for all code, terminal output, and token amounts
- ❌ **Don't**: Use Orbitron for body text or long paragraphs
- ❌ **Don't**: Mix more than 2 fonts in a single component

## 4. Logo Usage

### Logo Variations
1. **Full Logo**: Icon + "SolFoundry" wordmark (Orbitron)
2. **Icon Only**: Forge/anvil mark for small contexts (favicons, app icons)
3. **Wordmark Only**: "SolFoundry" text for inline references

### Clear Space
Maintain minimum clear space equal to the height of the "S" in the wordmark on all sides.

### Minimum Size
- Full logo: 120px width minimum
- Icon only: 24px minimum
- Wordmark: 80px width minimum

### Logo Rules
- ✅ **Do**: Use the full logo on dark backgrounds (`forge-950` or `forge-900`)
- ✅ **Do**: Use emerald or white for the logo on dark backgrounds
- ❌ **Don't**: Place the logo on busy backgrounds without a container
- ❌ **Don't**: Stretch, rotate, or add effects to the logo
- ❌ **Don't**: Use the logo smaller than minimum size
- ❌ **Don't**: Change the logo colors outside approved palette

## 5. Imagery & Graphics

### Style
- **Dark-themed**: All imagery should work on forge-950 backgrounds
- **Technical**: Terminal screenshots, code snippets, architectural diagrams
- **Glowing accents**: Use emerald/purple/magenta glow effects for emphasis
- **Grid patterns**: Subtle forge grid background for texture

### Gradients

| Name | Definition | Usage |
|------|-----------|-------|
| Navbar Gradient | `emerald → purple → magenta` | Navigation accent line |
| Hero Gradient | `Radial: purple center → magenta → transparent` | Hero background |
| Footer Gradient | `magenta → purple → emerald` | Footer accent line |

### Icons
- Use **Lucide React** icon set for consistency
- Icon size: 16px (`w-4 h-4`) for inline, 20px (`w-5 h-5`) for standalone
- Icon color: `text-muted` default, `text-emerald` for active/primary

## 6. Tone of Voice

### Personality
- **Direct**: Say what you mean. No corporate fluff.
- **Technical**: Speak developer language. Use code examples.
- **Encouraging**: Celebrate contributions. Reward is central.
- **Confident**: We know this works. No hedging.

### Voice Examples

| ✅ Do | ❌ Don't |
|-------|---------|
| "Submit your PR. Earn $FNDRY." | "We would love it if you could consider submitting..." |
| "Your code was reviewed and approved." | "We are pleased to inform you that upon careful review..." |
| "Bug found? Fix it. Get paid." | "We encourage community members to identify and resolve..." |
| "Fork → Code → PR → Earn" | "The contribution process involves several steps..." |

### Writing Guidelines
- Use **active voice**: "Submit your PR" not "PRs should be submitted"
- Keep sentences **short**: Under 20 words when possible
- Use **code formatting** for technical terms: `bounty`, `$FNDRY`, `PR`
- Numbers over amounts: "500K $FNDRY" not "five hundred thousand tokens"

## 7. Design Principles

### 1. Forge Aesthetic
Everything should feel like a digital workshop — dark, precise, industrial. Surfaces are dark steel. Accents glow like molten metal.

### 2. Clarity Over Decoration
Every element serves a purpose. Remove anything that doesn't help the user complete their task. White space is a feature, not wasted space.

### 3. Reward is Visual
Token amounts, tier badges, and bounty values should always be prominent. Use `JetBrains Mono`, emerald color, and generous sizing for reward displays.

### 4. Progressive Disclosure
Show essential information first. Details expand on interaction. Don't overwhelm with everything at once.

### 5. Accessible by Default
- Minimum contrast ratio: 4.5:1 for text, 3:1 for large text
- All interactive elements have focus indicators
- Animations respect `prefers-reduced-motion`
- Semantic HTML and ARIA labels

## 8. Component Patterns

### Buttons
- **Primary**: Emerald background, forge-950 text, rounded-lg
- **Secondary**: Forge-800 background, text-secondary, border
- **Ghost**: Transparent background, text-muted, hover: text-primary

### Cards
- Background: `forge-900`
- Border: `border` color
- Border radius: `rounded-xl`
- Padding: `p-5`
- Hover: `border-hover` color, subtle scale

### Badges / Pills
- Border radius: `rounded-full`
- Padding: `px-3 py-1`
- Font: `text-xs font-medium`
- Colors match tier or status
