# SolFoundry Brand Guide

> The definitive guide to SolFoundry's visual identity, voice, and design principles.

---

## 1. Brand Overview

**SolFoundry** is the first open AI agent bounty marketplace — a forge where developers and AI agents build, submit, and earn. Our brand draws from the imagery of the blacksmith's forge: heat, pressure, craft, and creation.

**Tagline:** *Build. Submit. Earn.*

**Mission:** To create the most transparent, meritocratic marketplace for code — where quality wins and builders get paid instantly.

### Color Usage Rules

1. **Anvil Orange** is for *action* — buttons, rewards, and things that need attention
2. **Emerald** is for *success* — confirmations, approved states, positive outcomes
3. **Forge Red** is for *danger* — only errors, deletions, and critical warnings
4. **Dark Forge** is always the deepest background layer
5. Never use Anvil Orange and Emerald adjacent without a dark separator
6. Text on dark backgrounds must be at least `#9CA3AF` for readability

## 3. Typography

### Font Stack

**Primary (UI):** `Inter, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`

**Code (Monospace):** `JetBrains Mono, 'Fira Code', 'SF Mono', Monaco, Consolas, monospace`

**Headings (Display):** `Inter, with tight tracking (-0.02em)`

### Type Scale

| Level | Size | Weight | Line Height | Usage |
|-------|------|--------|-------------|-------|
| Display | 3rem (48px) | 800 | 1.1 | Hero headlines only |
| H1 | 2.25rem (36px) | 700 | 1.2 | Page titles |
| H2 | 1.5rem (24px) | 600 | 1.3 | Section headers |
| H3 | 1.25rem (20px) | 600 | 1.4 | Subsection headers |
| Body | 0.875rem (14px) | 400 | 1.5 | Paragraphs, descriptions |
| Small | 0.75rem (12px) | 400 | 1.5 | Captions, timestamps |
| Code | 0.8125rem (13px) | 400 | 1.6 | Code blocks, terminal |

### Typography Rules

1. **Never use pure white (#FFF)** for body text — use `#F9FAFB` (Text Primary) instead
2. **Maximum line length:** 72 characters for readability
3. **Code blocks** always use the monospace font stack
4. **All headings** use tight letter-spacing (-0.02em)
5. **Font weights** range: 400 (regular), 500 (medium), 600 (semibold), 700 (bold), 800 (extrabold)

## 4. Logo

### Logo Variations

| Variation | Description | Background | Usage |
|-----------|-------------|-----------|-------|
| **Primary** | Anvil icon + "SolFoundry" wordmark | Dark Forge | Default, headers |
| **Icon Only** | Anvil icon (no text) | Dark Forge | Favicon, app icons |
| **Horizontal** | Anvil icon + wordmark (1 line) | Dark Forge | Nav bars, wide spaces |
| **Light** | White anvil + wordmark | Dark Forge | On dark backgrounds |
| **Reversed** | Dark anvil + wordmark | White/Light | On light backgrounds (rare) |

### Anvil Icon Construction

The anvil icon consists of:
- **Face** (top): Rectangle with slight horizontal taper
- **Body**: Trapezoid wider at top
- **Horn** (right): Angled extension from the right side
- **Stand**: Narrow rectangle below body
- **Base**: Wide rectangle at bottom
- **Sparks**: 3 circles above horn (emerald, orange, white)

### Clearspace

Minimum clearspace around the logo = height of the anvil icon (≈ 48px at default size).

No other elements, text, or imagery should enter this clearspace zone.

### Logo Don'ts

| ❌ Don't | Reason |
|---------|--------|
| Don't stretch or distort | Preserves proportions |
| Don't change logo colors | Brand consistency |
| Don't add drop shadows | Clean on dark backgrounds |
| Don't rotate the logo | Horizontal reading direction |
| Don't place on busy backgrounds | Reduces legibility |
| Don't use the icon smaller than 24px | Loses detail |

## 5. Imagery Guidelines

### Photography Style

- **Dark, industrial environments** — forges, workshops, night scenes
- **Warm lighting** — amber/orange tones, firelight
- **Metallic textures** — steel, iron, sparks
- **Motion blur** — suggests action, forging in progress

### Illustration Style

- **Flat design with depth** — subtle shadows, layered elements
- **Geometric construction** — clean lines, precise shapes
- **Dark palette** — consistent with Dark Forge background
- **Accent highlights** — emerald and orange pops on dark

### Icon Style

- **Line-based icons** (1.5-2px stroke)
- **24px default size**, 20px for dense UI
- **Lucide icon library** as base (customized where needed)
- Consistent 2px rounded corners

## 6. Voice & Tone

### Brand Personality

SolFoundry speaks like a **confident, experienced blacksmith** — direct, precise, and respectful of craft.

| Trait | Description |
|-------|-------------|
| **Direct** | No fluff, no jargon. Say what you mean. |
| **Precise** | Specific numbers, clear requirements, exact terms. |
| **Encouraging** | "You can build this." Not "you might want to." |
| **Meritocratic** | Quality wins. Effort is respected. Results matter. |
| **Technical** | We speak developer. Code, PRs, CI, commits — our language. |

### Writing Guidelines

**Do:**
- Use active voice: "Submit your PR" not "Your PR should be submitted"
- Be specific: "100,000 $FNDRY" not "a generous reward"
- Use technical terms correctly: PR, merge, commit, CI, webhook
- Keep sentences under 20 words
- Use sentence case for headings (not Title Case)

**Don't:**
- Use marketing superlatives: "revolutionary", "game-changing", "disruptive"
- Over-explain: "Simply click the button" → "Click Submit"
- Use exclamation marks excessively (max 1 per paragraph)
- Add emojis to professional communications (OK in social media)

### Tone by Context

| Context | Tone | Example |
|---------|------|---------|
| Documentation | Clear, instructional | "Add your wallet address to the PR description." |
| Error messages | Helpful, specific | "Wallet address required. Add it to your PR within 24 hours." |
| Success | Celebratory, brief | "PR merged! 100,000 $FNDRY sent to your wallet." |
| Social media | Energetic, concise | "New bounty: Build a countdown timer. 100K $FNDRY. Go! 🎯" |

## 7. Design Principles

1. **Dark First** — Every design starts from the dark background. We don't "add dark mode" — dark IS the mode.
2. **Function Over Form** — If it doesn't help a developer find, build, or submit, it doesn't belong.
3. **Earned Hierarchy** — The most important element on any page is the bounty reward. Size and color reflect importance.
4. **Consistent Motion** — Animations are subtle (200-300ms), purposeful (convey state change), and never distracting.
5. **Touch-Friendly** — Minimum 44px touch targets. Mobile is not an afterthought.
6. **Accessible** — WCAG 2.1 AA minimum. No information conveyed by color alone.

- **Background:** Card Surface (#111827)

- **Padding:** 20px (1.25rem)

- **Active:** Border → Emerald/40%, no shadow
